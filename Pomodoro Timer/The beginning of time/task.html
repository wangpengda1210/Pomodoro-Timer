About

All of us sometimes want to study or work, but then something truly mysterious happens: first, we get distracted by a message from a friend, then we decide to briefly check social media, where we bump into some breaking news. Three hours later we find ourselves reading an article about the fall of the Roman Empire. It might be hard to finally stop procrastinating and spend time on really important things, and this is where time management comes in handy. One of the most famous planning techniques is Pomodoro.

Pomodoro consists of six steps:

  1. Choose the task.
  2. Set the pomodoro timer (typically for 25 minutes).
  3. Work on the task.
  4. Stop working when the timer goes off and put a checkmark on a piece of paper.
  5. If you have fewer than four checkmarks, take a short break (3–5 minutes), then go to step 2.
  6. After four pomodoros (that is, checkmarks), take a longer break (15–30 minutes), reset your checkmark count to zero, then go back to step 1.

Quite straightforward and helpful, isn’t it? By the end of this project, you will be able to implement an Android mobile application with a Pomodoro timer.

To save time and simplify the testing process, the app will work with seconds instead of minutes.

In this stage, you should create and run a simple application with Android UI components (Views).

Android Features

Firstly, you can consider these Android-specific topics:

Android Studio (https://developer.android.com/studio/intro) is our main IDE for Android development. This link takes you to a short introduction to functions and shortcuts. If you have already used Intellij idea, many features will be familiar to you. The main difference is launching an application. You may find information about it in the Instruction at the end of the document.

Android API (http://socialcompare.com/en/comparison/android-versions-comparison): same as all frameworks and OS’s, Android has versions. Versions of APIs relate to features added (for instance, in 2.3.3 NFC support was added): the higher the version, the more features it has. The choice of the version is really important because some components in both UI and code might not be supported in older versions. This link shows a table of Android versions and descriptions of features added in each version.

Android SDK (https://www.androidauthority.com/android-sdk-tutorial-beginners-634376/): SDK (Software Development Kit) is pretty similar to JDK, but tailored for Android needs. JDK has a compiler and SDK has build tools. JDK has a debugger and SDK has the Android Debug Bridge (ADB). However, we may run a Kotlin or Java app through the console, while for Android development we have to use emulators that are provided by SDK. If you follow this link, you'll find a description of the components of SDK.

View, Layout, XML (https://google-developer-training.github.io/android-developer-fundamentals-course-concepts-v2/unit-1-get-started/lesson-1-build-your-first-app/1-2-c-layouts-and-resources-for-the-ui/1-2-c-layouts-and-resources-for-the-ui.html): in Swing (or in any another UI framework), we have UI components like JCheckBox, JSlider etc. In Android development, those are called Views. If you want to build beautiful and user-friendly UIs, you should know about material design (https://developer.android.com/design) and representation of this approach via Layouts. This link describes Layouts as components for arranging Views on the screen and explains how to declare Views in XML, set up attributes and use them in the code.

Description

In this stage, you should create two buttons and one label.

To display, start, and reset the timer, we need several components (Views) on the screen of the app. You should set up the environment (Android Studio and Virtual Device\Real Device), create a project with an empty activity, and put the following on the screen of this activity:

  1. A button with attributes: text: Start, id: startButton.
  2. A button with attributes: text: Reset, id: resetButton.
  3. A TextView with attributes: id: textView.

All these Views should be placed inside MainActivity class with the layout named activity_main.

Remember to set id's to these components because then we would be able to test them!
